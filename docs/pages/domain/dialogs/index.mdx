# Domain Dialogs

Business dialogs for Edge Platform management operations. These are Layer 3 components — they accept props only and do not call APIs directly.

## Layer Rules

| Layer | Package | Responsibility |
|-------|---------|----------------|
| **Layer 2** | `@edge/ui` | Generic dialog patterns: `ConfirmDeleteDialog`, `NameConfirmDeleteDialog`, `CreateResourceDialog`, `YamlEditDialog` |
| **Layer 3** | `@edge/components` | Business dialogs — accept props only, no API calls |

## Common Pattern

All business dialogs follow this pattern:

```tsx
interface XxxDialogProps {
  open: boolean                          // or isOpen
  onOpenChange: (open: boolean) => void  // or onClose
  // ... resource-specific props
  onConfirm: (...) => Promise<void>      // async callback, no API call inside dialog
  isLoading?: boolean                    // optional external loading state
}
```
